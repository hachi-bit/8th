# 01. 床置きオブジェクト配置デモ 仕様書

> 前提: `00-common-setup.md` の内容を先に読み、共通ルールに従うこと。

## 1. 目的

8th WallのWorld Effects(SLAM)による平面検出の基本性能を検証する、最もシンプルなデモ。「タップした場所にオブジェクトが実寸で置け、歩いて周囲から見られるか」を確認する。

## 2. 成功の定義

- ユーザーが画面をタップすると、その位置(平面/特徴点)に3Dオブジェクトが配置される
- 配置したオブジェクトは実世界に固定されて見え、スマホを動かしても位置がずれない(いわゆる「浮いて追従してしまう」不具合が起きない)
- 複数回タップすれば複数個配置できる

## 3. 機能要件

### 3.1 配置オブジェクト
- 一辺20cm(`0.2` three.js単位 = メートル)の立方体(`THREE.BoxGeometry(0.2, 0.2, 0.2)`)
- マテリアルは`THREE.MeshStandardMaterial`、色は`#2f8fff`程度の視認性の良い青
- 配置位置は、ヒットテストで得たY座標に立方体の半分の高さ(0.1m)を足した位置に置く(立方体が床に埋まらないようにする)

### 3.2 タップ操作
- `00-common-setup.md` 3.4節の共通ヒットテスト関数を使用する
- ヒットテストが失敗した場合(`null`が返った場合)、ステータステキストに「平面が検出されへんかった。カメラをゆっくり動かして周囲をスキャンしてみて」のように、動かして再スキャンを促す文言を表示する
- 配置成功時は「配置済み: N個」のようにカウントを表示する

### 3.3 Recenterボタン
- 画面右下に「Recenter」ボタンを設置
- 押下時に `XR8.XrController.recenter()` を呼び出す
- トラッキングが乱れた/おかしくなったと感じたユーザーがリセットする用途

### 3.4 ライティング
- `THREE.AmbientLight(0xffffff, 0.7)` で環境光
- `THREE.DirectionalLight(0xffffff, 0.6)` を `(1, 3, 1)` 付近に配置し、立体感を出す

## 4. UI要素

| 要素 | 内容 | 位置 |
|---|---|---|
| ヒント文 | 「床(平面)をタップすると、その場所にオブジェクトが置かれるのら。歩いて周りから見てみてや」 | 画面上部 |
| ステータス | 配置数 or エラー案内 | 画面下部中央 |
| Recenterボタン | リセンター実行 | 画面右下 |

## 5. ファイル名

`1-floor-placement.html`(単一ファイル、CSS/JSインライン)

## 6. 受け入れ基準

- [ ] タップした場所に立方体が出現する
- [ ] 立方体が床に埋まらず、浮きすぎもしない(±数cm程度の誤差は許容)
- [ ] 5個程度連続で配置してもフレームレートが目に見えて落ちない
- [ ] Recenterボタン押下後もアプリがクラッシュせず継続動作する
- [ ] `00-common-setup.md`の共通受け入れ基準をすべて満たす

## 7. 拡張候補(今回のスコープ外、実装しない)

- 配置したオブジェクトの削除・選択機能
- オブジェクトの種類切り替え(立方体以外)
- ARShadow(疑似影)の追加

これらは検証の主目的(平面検出の基本性能確認)に直接関係しないため、今回は実装しないこと。ユーザーから追加依頼があった場合のみ着手する。
