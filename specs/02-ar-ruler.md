# 02. ARバーチャル巻尺デモ 仕様書

> 前提: `00-common-setup.md` の内容を先に読み、共通ルールに従うこと。

## 1. 目的

8th WallのWorld Tracking(SLAM)によって得られる実世界スケールの精度を検証する。2点タップでその間の実寸距離をメートル表示する。

## 2. 成功の定義

- ユーザーが2点をタップすると、その2点間の直線距離がメートル単位(小数点2桁)で表示される
- 既知の長さのもの(例: A4用紙の長辺29.7cm、1mの物差し等)を実際に測って、誤差が実用上許容できる範囲(目安: ±5%程度)に収まるかを実機検証する

## 3. 機能要件

### 3.1 計測フロー(状態遷移)

1. **初期状態**: ステータス「1点目をタップしてね」
2. **1点目タップ**: 赤色(`#ff4d4d`)の小球(半径0.03m)をその位置に配置。ステータスを「2点目をタップしてね」に変更
3. **2点目タップ**: 青色(`#4d94ff`)の小球を配置。2点間に緑色(`#7CFC9A`)の直線(`THREE.Line`)を描画。2点のユークリッド距離を計算し、画面下部に大きく表示(例: `1.85 m`)
4. **3回目のタップ**: 既存の計測(マーカー2つ・線・距離表示)をすべてクリアし、状態1に戻る(新しい計測を開始)
5. **「リセット」ボタン**: いつでも状態1に戻せる

### 3.2 距離計算

3D空間上の2点 `p1 = {x,y,z}`, `p2 = {x,y,z}` に対し、単純なユークリッド距離を計算する:

```javascript
const dx = p1.x - p2.x
const dy = p1.y - p2.y
const dz = p1.z - p2.z
const meters = Math.sqrt(dx*dx + dy*dy + dz*dz)
```

Y軸(高さ)方向の差も含めた3次元距離であることに注意。床面上の2点を測る想定だが、高さが異なる2点(例: 机の上と床)を測っても正しく動作すること。

### 3.3 ヒットテスト失敗時

`00-common-setup.md`の共通ヒットテスト関数が`null`を返した場合、「平面/特徴点が検出できませんでした。もう一度試してください」とステータス表示し、タップ状態は変化させない(1点目待ちなら1点目待ちのまま)。

## 4. UI要素

| 要素 | 内容 | 位置 |
|---|---|---|
| ヒント文 | 「2点をタップして実寸の距離を測定します(1点目→2点目)」 | 画面上部 |
| 距離表示 | 計測結果を大きな緑文字で表示(例: `1.85 m`) | 画面下部やや上 |
| ステータス | タップ待ちの状態案内 | 画面下部 |
| リセットボタン | 計測をクリアして最初からやり直す | 画面右下 |

## 5. ファイル名

`2-ar-ruler.html`(単一ファイル、CSS/JSインライン)

## 6. 受け入れ基準

- [ ] 2点タップで距離がメートル単位で表示される
- [ ] 既知の長さの実物(30cm定規など)を測定し、表示された値をログか手元メモに残す(実機検証の記録として)
- [ ] 3回目のタップで正しくリセットされ、古いマーカー・線が画面に残り続けない(three.jsのシーンからの`remove`漏れがないこと)
- [ ] リセットボタンでも同様にクリアされる
- [ ] `00-common-setup.md`の共通受け入れ基準をすべて満たす

## 7. 既知の注意点

- Absolute Scale機能(8th Wallのバイナリ限定機能)は本デモでは**使用しない**。World Tracking(SLAM)が提供する標準スケールのみで実装すること。Absolute Scaleを使った高精度版は将来の拡張候補とし、今回のスコープには含めない。
- 精度検証の結果次第で、将来的にAbsolute Scaleへの切り替えを検討する可能性がある旨をコード内コメントに残しておくこと。
