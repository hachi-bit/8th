# 03. 提案評価タワー AR配置デモ 仕様書

> 前提: `00-common-setup.md` の内容を先に読み、共通ルールに従うこと。
> 本デモは既存の「Three.js島型シティスケープ評価ツール」(コスト/価値・品質・スケジュール・実績の4軸で契約者を評価する既存プロジェクト)の、簡易AR版プロトタイプである。

## 1. 目的

既存の提案評価ツールの評価軸データを、実寸大のARオブジェクトとして現実空間に配置できるかを検証する。評価担当者が実際にその場を歩き回りながら評価結果を確認できる体験の実現可能性を見る。

## 2. 成功の定義

- 床をタップすると、4つの評価軸を積み上げた実寸大の「評価タワー」が1本その場に生成される
- タワーの周りを歩いて、上から下まで見て回れる
- 各軸のスコアがタワーの各セグメントの高さに反映されている

## 3. データ構造

サンプルスコアをコード内にハードコードする(実データ連携は本デモのスコープ外)。0〜1の範囲の数値とする。

```javascript
const SCORES = {
  cost:     0.7,  // コスト/価値
  quality:  0.85, // 品質
  schedule: 0.55, // スケジュール
  track:    0.9,  // 実績
}
const COLORS = {
  cost:     0x4d94ff, // 青
  quality:  0x7CFC9A, // 緑
  schedule: 0xffb84d, // 橙
  track:    0xff6b6b, // 赤
}
```

この配色は既存の島型シティスケープ評価ツールとの一貫性を意識したものなので、既存プロジェクトの配色ルールがあればそちらを優先し、このドキュメントの色は仮値として扱ってよい。

## 4. タワーの構造(下から上へ)

1. **台座**: 半径0.19〜0.22m程度の薄い円柱(`THREE.CylinderGeometry`)、色はダークグレー(`0x333333`)、高さ0.02m
2. **4軸セグメント**: `cost → quality → schedule → track` の順に下から積み上げる。各セグメントは半径0.12mの円柱。高さはスコアに比例させる(全体の最大高さ`TOWER_MAX_HEIGHT = 1.2m`を4軸の満点合計で割った単位に、各スコアを掛けた値)
3. **総合スコア球**: タワーの最上部から0.1m浮かせた位置に配置。半径は`0.06 + overall * 0.05`(overallは4軸の平均値)で、スコアが高いほど大きくなる。色は白(`0xffffff`)、`emissive: 0x333333`でわずかに発光させ、頂上の目印として視認性を確保する

## 5. 機能要件

### 5.1 配置

- `00-common-setup.md`の共通ヒットテスト関数でタップ位置を取得し、そこにタワー全体(`THREE.Group`)を配置する
- **本デモでは1個だけ配置可能とする**(2回目以降のタップは無視する)。複数社比較のような複数配置機能は次フェーズの拡張候補とし、今回は実装しない
- 配置後、ステータスに「配置が完了しました。周りを歩いて見てみてください」と表示する

### 5.2 凡例(レジェンド)表示

画面左下に、4軸の色と名称の対応表を常時表示する:

```
■ コスト/価値   (青)
■ 品質          (緑)
■ スケジュール  (橙)
■ 実績          (赤)
```

### 5.3 Recenterボタン

`01-floor-placement.md`と同様に画面右下に設置し、`XR8.XrController.recenter()`を呼ぶ。

## 7. ファイル名

`3-proposal-tower-ar.html`(単一ファイル、CSS/JSインライン)

## 8. 受け入れ基準

- [ ] 床タップでタワーが1本、実寸大(最大1.2m程度)で出現する
- [ ] 4色のセグメントが下から`cost → quality → schedule → track`の順で積まれている
- [ ] 頂上の総合スコア球が視認できる
- [ ] 2回目以降のタップで新しいタワーが増えない(仕様通り1個のみ)
- [ ] タワーの周囲を歩いても位置がずれず、床に固定されて見える
- [ ] `00-common-setup.md`の共通受け入れ基準をすべて満たす

## 9. 将来の拡張候補(今回のスコープ外)

- 実際の評価データ(既存Three.js島型シティスケープツールのデータ)との連携
- 複数社のタワーを同時に配置して比較する機能
- タワーをタップすると詳細スコアがポップアップ表示される機能
- 会議室などで複数人が同じタワーを共有して見るマルチユーザーAR(8th Wallの現行の無償構成では非対応。将来的にVPS等の代替手段が必要になる可能性がある点をコード内コメントに残しておくこと)
